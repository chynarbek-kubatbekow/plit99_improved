## Быстрый старт

### 1. Установка зависимостей

```bash
pip install django pillow
```

### 2. Применить миграции

```bash
python manage.py migrate
```

### 3. Заполнить тестовыми данными

```bash
python manage.py seed_data
```

### 4. Создать администратора

```bash
python manage.py createsuperuser
```

### 5. Запустить сервер

```bash
python manage.py runserver
```

Сайт будет доступен по адресу: **http://127.0.0.1:8000/**

## Административная панель

**URL:** http://127.0.0.1:8000/admin/

**Тестовые данные для входа (если использовалась seed_data):**
- Создайте пользователя командой `createsuperuser`

### Что можно делать в админке:

#### 📰 Новости
- Добавлять / редактировать / удалять новости
- Загружать фото-обложку или задавать эмодзи + градиент
- Назначать категории и метку «Главная новость»
- Управлять публикацией (черновик / опубликовано)
- Фильтрация по дате, категории, статусу
- После сохранения контент автоматически дублируется в JSON snapshot для публичного fallback

#### 🖼️ Галерея
- Загружать фотографии
- Добавлять видео (ссылка на YouTube/Vimeo)
- Сортировать по категориям (Мероприятия, Учёба, Спорт, Международное)
- Устанавливать порядок отображения
- Фотографии сохраняются в `media/`, а публичный контент дублируется в JSON snapshot

## Отказоустойчивость контента

Для новостей и галереи проект теперь использует двойное хранение:

- основное: Django ORM / база данных
- резерв: JSON snapshot + `media`

При сохранении через админку:

- записи сохраняются в БД
- изображения попадают в `media`
- публичный контент экспортируется в `site_content.json`

Если база данных временно недоступна, публичные страницы новостей и галереи читают данные из snapshot-файла и продолжают работать.

## Переход на PostgreSQL (продакшн)

В `settings.py` замените:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'plit99_db',
        'USER': 'plit99_user',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
}
```
